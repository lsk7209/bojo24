# Gemini API 활용 제안서

## 📋 현재 상황 분석

### 문제점
- 공공데이터에서 제공되는 정보가 적어서 핵심요약, 지원대상, 지원내용에 보이는 내용의 양이 적음
- 공공데이터만으로는 사용자에게 충분한 정보를 제공하기 어려운 경우가 많음

### 해결 방안: 하이브리드 접근
**공공데이터를 기반으로 하되, 부족한 부분은 Gemini로 보완**

---

## 🎯 Gemini API 활용 추천 영역

### 1. **핵심 요약 (Summary) - 최우선 추천** ⭐⭐⭐

**현재 문제:**
- 공공데이터의 서비스목적, 지원대상, 지원내용이 짧거나 불완전
- 단순 나열식으로 가독성이 떨어짐

**Gemini 활용 방안:**
```typescript
// 공공데이터가 충분하면 공공데이터 사용
// 공공데이터가 부족하면 (200자 미만) Gemini로 확장
if (summary.length < 200 || !hasEnoughInfo(detail)) {
  const enhancedSummary = await generateEnhancedSummary(benefit, detail);
  // Gemini가 공공데이터를 분석하여:
  // - 사용자 관점에서 재구성
  // - 구체적인 예시 추가
  // - 실용적인 정보 강조
}
```

**생성 내용:**
- 공공데이터를 기반으로 한 **분석적 요약** (단순 나열 X)
- 사용자 시나리오 기반 설명
- 구체적인 금액, 기간, 혜택 강조
- "누가", "무엇을", "얼마나", "언제" 명확히 정리

**구글 고유 컨텐츠 인정 요소:**
- ✅ 공공데이터 기반 (출처 명시)
- ✅ AI 분석/해석 추가 (가치 창출)
- ✅ 사용자 관점 재구성 (고유성)

---

### 2. **지원 대상 (Target) - 추천** ⭐⭐

**현재 문제:**
- 공공데이터의 지원대상이 "○ 은평구 거주 중증장애인"처럼 간략함
- 구체적인 자격 요건, 예외 사항이 명시되지 않음

**Gemini 활용 방안:**
```typescript
// 공공데이터가 간략하면 (100자 미만) Gemini로 구체화
if (target.length < 100) {
  const enhancedTarget = await generateEnhancedTarget(benefit, detail);
  // Gemini가:
  // - 자격 요건을 구체적으로 설명
  // - 예외 사항 명시
  // - 실제 신청 가능한 사람들의 예시 제공
}
```

**생성 내용:**
- 자격 요건 구체화 (나이, 소득, 거주지 등)
- 예외 사항 명시
- 실제 신청 가능한 사람들의 예시
- "이런 분들은 신청 가능합니다" 형태로 친근하게 설명

**구글 고유 컨텐츠 인정 요소:**
- ✅ 공공데이터 기반 해석
- ✅ 실용적인 예시 추가
- ✅ 사용자 이해도 향상

---

### 3. **지원 내용 (Benefit) - 추천** ⭐⭐

**현재 문제:**
- 공공데이터의 지원내용이 "○ 장애인 편의시설 설치 지원"처럼 간략함
- 구체적인 금액, 혜택 범위, 사용 방법이 불명확

**Gemini 활용 방안:**
```typescript
// 공공데이터가 간략하면 Gemini로 상세화
if (benefit.length < 150) {
  const enhancedBenefit = await generateEnhancedBenefit(benefit, detail);
  // Gemini가:
  // - 구체적인 금액/혜택 범위 명시
  // - 사용 방법 설명
  // - 실제 받을 수 있는 혜택의 예시 제공
}
```

**생성 내용:**
- 구체적인 금액/혜택 범위 명시
- 사용 방법 및 제한 사항
- 실제 받을 수 있는 혜택의 예시
- "이렇게 활용하실 수 있습니다" 형태로 실용적 설명

**구글 고유 컨텐츠 인정 요소:**
- ✅ 공공데이터 기반 확장
- ✅ 실용적인 가이드 제공
- ✅ 사용자 가치 창출

### 4. **추가 설명 섹션 (Optional) - 선택적** ⭐

**현재 문제:**
- 공공데이터에는 없는 맥락 정보가 부족
- "왜 이 정책이 필요한가?", "어떤 효과가 있는가?" 등

**Gemini 활용 방안:**
```typescript
// 선택적으로 추가 설명 생성
const contextExplanation = await generateContextExplanation(benefit, detail);
// Gemini가:
// - 정책 배경 설명
// - 사회적 의미 설명
// - 유사 정책과의 차이점
```

**생성 내용:**
- 정책 배경 및 목적 설명
- 사회적 의미 및 효과
- 유사 정책과의 차이점
- "이 정책이 왜 필요한가요?" 형태의 설명

**구글 고유 컨텐츠 인정 요소:**
- ✅ 분석적 관점 제공
- ✅ 맥락 정보 추가
- ✅ 전문성 강조

## 🔧 구현 전략

### 하이브리드 접근 방식

```typescript
// 1. 공공데이터 우선 사용
let summary = generateSummaryFromPublicData(detail);

// 2. 공공데이터가 부족하면 Gemini로 보완
if (needsEnhancement(summary, detail)) {
  const enhanced = await enhanceWithGemini(benefit, detail);
  summary = mergePublicDataAndAI(enhanced, summary);
}

// 3. 항상 공공데이터 출처 명시
summary += "\n\n[출처: 행정안전부 보조24 공공데이터]";
```

### 품질 기준

**Gemini 보완이 필요한 경우:**
- 핵심 요약: 200자 미만
- 지원 대상: 100자 미만
- 지원 내용: 150자 미만
- 또는 필수 정보(금액, 기간 등)가 누락된 경우

**Gemini 보완 불필요한 경우:**
- 공공데이터가 이미 충분히 상세함
- 모든 필수 정보가 포함되어 있음

## 🚀 구현 우선순위

1. **1단계: 핵심 요약 보완** (최우선)
   - 공공데이터가 부족한 경우 Gemini로 확장
   - 사용자 관점으로 재구성

2. **2단계: 지원 대상/내용 보완**
   - 간략한 공공데이터를 구체화
   - 실용적인 예시 추가

3. **3단계: 추가 설명 섹션** (선택적)
   - 정책 배경 및 맥락 설명
   - 분석적 관점 제공

## ⚠️ 주의사항

1. **출처 명시 필수**

   - 캐싱 전략 활용
