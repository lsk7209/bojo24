# 보조금 페이지 재생성 가이드

## 재생성할 페이지 ID
- 318000000430
- 371000000106
- 394000000138

## 방법 1: Vercel 환경 변수 설정 (권장)

1. Vercel 대시보드 접속
2. 프로젝트 선택 → Settings → Environment Variables
3. `GEMINI_ENHANCEMENT_ALLOWED_IDS` 변수 수정:
   - 기존 값이 있다면: `기존값,318000000430,371000000106,394000000138`
   - 없다면 새로 생성: `318000000430,371000000106,394000000138`
4. 배포 (자동 또는 수동)
5. 해당 페이지 방문:
   - https://www.bojo24.kr/benefit/보건·의료/318000000430
   - https://www.bojo24.kr/benefit/생활안정/371000000106
   - https://www.bojo24.kr/benefit/임신·출산/394000000138

## 방법 2: 로컬에서 실행

`.env` 파일에 다음 변수 추가:

```env
GEMINI_API_KEY=your_gemini_api_key
GEMINI_ENHANCEMENT_ALLOWED_IDS=318000000430,371000000106,394000000138
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

그 다음 실행:

```bash
npm run regenerate:benefits
```

## 재생성 내용

각 페이지의 다음 섹션이 동적 글자수 조정과 함께 재생성됩니다:

1. **핵심 요약** - 현재 글자수에 따라 200~800자로 조정
2. **지원 대상** - 현재 글자수에 따라 200~800자로 조정
3. **지원 내용** - 현재 글자수에 따라 200~800자로 조정
4. **신청 방법** - 현재 글자수에 따라 200~800자로 조정
5. **필요서류** - 줄바꿈 오류 수정 및 정리
6. **FAQ 답변** - 현재 글자수에 따라 200~800자로 조정

## 확인 방법

재생성 후 페이지를 방문하여 다음을 확인하세요:

1. 각 섹션의 글자수가 증가했는지
2. 가독성이 향상되었는지 (볼드, 목록 등)
3. 예시가 포함되었는지 (지원 대상, 지원 내용)
4. 필요서류가 정리되었는지
